אוטומציה של פריסת אתר אישי באמצעות CI/CD

פרויקט זה נועד להפוך את תהליך הפריסה של האתר האישי שלי לאוטומטי לחלוטין. במקום להעלות קבצים באופן ידני לשרתים, בניתי תהליך עבודה (Workflow) של CI/CD, שמתחיל אוטומטית בכל פעם שמתבצע שינוי בקוד.

האתגר

לפני הפרויקט, כל שינוי בקוד דרש התחברות לשרת, מחיקת קבצים ישנים והעלאת קבצים חדשים. תהליך זה היה ארוך, חשוף לטעויות ולא איפשר סביבת פיתוח מהירה.

הפתרון

הקמתי צינור (Pipeline) אוטומטי באמצעות GitHub Actions ושירותי AWS. התהליך כלל את השלבים הבאים:

  • GitHub Repository: הקוד של האתר נמצא במאגר ב-GitHub.
  • GitHub Actions: הגדרתי קובץ `deploy.yml` שמשמש כ-Workflow. קובץ זה מופעל אוטומטית בכל פעם שקוד נדחף לענף הראשי (`master`).
  • AWS S3: יצרתי Bucket ב-S3 לאחסון האתר הסטטי. ה-Bucket הוגדר לאירוח אתרים, ובוצעו בו הגדרות אבטחה ופרמטרים של הפצת תוכן (CDN).
  • AWS IAM: יצרתי משתמש מוגבל הרשאות ב-AWS עם מפתח גישה (Access Key) וסוד גישה (Secret Access Key), שניתן לו רק הרשאות לפרוס קבצים ל-Bucket הספציפי.
  • Secrets: מפתחות הגישה של AWS הוגדרו כ-Secrets ב-GitHub Actions, כדי לשמור עליהם באופן מאובטח.

התוצאה

התוצאה היא תהליך פריסה מהיר, מאובטח ואוטומטי. כעת, כל שינוי שאני מבצע בקוד נפרס לאתר תוך שניות, ללא צורך בהתערבות ידנית. הדבר מאפשר לי להמשיך לפתח פרויקטים נוספים ולשפר את האתר שלי בקלות וביעילות.
